# Project Context

Tech stack: Markdown specs, shell scripts (Bash), Claude Code skills
Content: Research analyses of SpecKit and OpenSpec, cross-cutting findings
Workflow: Fab kit for specification-driven change management
Structure: Monorepo with doc/, fab/ directories
Local backlog: fab/backlog.md
Distribution: Kit content (skills, templates, migrations) is distributed to the system cache at `~/.fab-kit/versions/<version>/kit/`. User projects contain `fab/` (project config, changes) and `.claude/skills/` (deployed copies). Kit content is NOT copied into user projects.
Skills: Canonical skill sources live at `src/kit/skills/*.md` in the dev repo. The files in `.claude/skills/` are deployed copies generated by `fab sync` from the system cache. When modifying a skill, always update the source at `src/kit/skills/` — never edit only the deployed copy, or the change will be overwritten on next sync.
Migrations: When a change requires restructuring existing user data (config files, `.status.yaml`, archive layout, etc.), it MUST ship as a migration file in `src/kit/migrations/` — not as a new subcommand or ad-hoc script. Migrations are markdown instruction files applied by `/fab-setup migrations`. See `docs/memory/fab-workflow/migrations.md` for the format and versioning model.
