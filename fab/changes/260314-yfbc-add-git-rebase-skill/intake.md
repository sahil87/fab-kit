## Why

The fab workflow frequently involves feature branches that need to stay current with main. Currently there's no skill to handle rebasing — users must manually run the git commands. A `/git-rebase` skill provides a safe, guided rebase with proper guards:

1. Prevents accidental rebase when on main/master (which would be a no-op or confusing)
2. Detects uncommitted changes and asks the user how to handle them before rebasing (stash or abort)
3. Provides clear conflict resolution guidance when rebase fails

Without this, users risk losing uncommitted work or running rebase in unexpected states.

## What Changes

### New skill file: `fab/.kit/skills/git-rebase.md`

A new skill that performs `git fetch origin main && git rebase origin/main` with safety guards.

**Step 1 — Git repo check**: Verify inside a git repository via `git rev-parse --is-inside-work-tree`.

**Step 2 — Branch guard**: Refuse to run on `main` or `master`. Reports error and stops.

**Step 3 — Uncommitted changes check**: Run `git status --porcelain`. If output exists, show the user their pending changes and offer two options:
1. Stash changes, rebase, then restore (`git stash push` → rebase → `git stash pop`)
2. Abort — cancel so user can commit or handle changes first

Uses `AskUserQuestion` tool to get user confirmation.

**Step 4 — Fetch and rebase**:
- Auto-detect main branch name (`main` vs `master`)
- `git fetch origin {main_branch}`
- `git rebase origin/{main_branch}`
- If stashing: `git stash push` before, `git stash pop` after
- On fetch failure with stash: pop stash before stopping
- On rebase conflict: report and advise on `git rebase --continue` or `git rebase --abort`

**Step 5 — Report**: Confirmation message with branch names and stash status.

**Allowed tools**: `Bash(git:*)`, `AskUserQuestion`
