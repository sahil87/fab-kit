## What's New in v1.7.0

### Decoupled wt and idea

`wt` and `idea` are now standalone Homebrew formulas. fab-kit's formula declares them as dependencies (`depends_on "sahil87/tap/wt"` and `depends_on "sahil87/tap/idea"`), so `brew install sahil87/tap/fab-kit` still installs all four CLIs (`fab`, `fab-kit`, `wt`, `idea`) on PATH transitively. Each binary now versions and releases independently from its own repository (`github.com/sahil87/wt`, `github.com/sahil87/idea`).

### Upgrade

For most users, `brew upgrade fab-kit` is sufficient — the standalone `wt` and `idea` formulas declare `link_overwrite "bin/wt"` and `link_overwrite "bin/idea"`, so they take ownership of the previously fab-kit-bundled symlinks silently.

**Troubleshooting fallback** (only needed in the rare case `link_overwrite` does not resolve cleanly):

```
brew unlink wt idea && brew upgrade fab-kit
```

This is a fallback, not a required pre-step. After upgrade, `wt --version` and `idea --version` will report each standalone formula's version (rather than the previously-bundled `fab-kit 1.6.2`).
