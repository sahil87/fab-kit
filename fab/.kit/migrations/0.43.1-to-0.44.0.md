# Migration: 0.43.1 to 0.44.0

## Summary

Transition to the system shim model. The `fab` command is now a system binary installed via Homebrew (`brew install fab-kit`) that provides version-aware dispatch. Binaries are removed from the repo; `fab/.kit/` contains content only.

## Pre-check

1. **Prerequisite gate**: Verify `fab` is on PATH (the system shim). Run `command -v fab` — if it fails, stop with:
   ```
   fab not found on PATH. Install the system shim first:
     brew tap wvrdz/tap && brew install fab-kit
   ```
2. Confirm `fab/.kit/VERSION` reads `0.44.0` or higher

## Changes

### 1. Add `fab_version` to `config.yaml`

If `fab/project/config.yaml` exists but does not contain a `fab_version` field:

1. Read the current kit version from `fab/.kit/VERSION`
2. Add `fab_version: "{version}"` as the first line of `config.yaml`
3. Print: `Added fab_version: {version} to config.yaml`

If `fab_version` already exists, print: `fab_version already set, skipping.`

### 2. Clean `.envrc`

If `.envrc` contains a line matching `PATH_add fab/.kit/bin`:

1. Remove that line from `.envrc`
2. Print: `Removed 'PATH_add fab/.kit/bin' from .envrc (fab is now on system PATH via Homebrew).`

If `.envrc` does not contain that line, print: `.envrc already clean, skipping.`

### 3. Clean `fab/.kit/bin/`

Remove the following files from `fab/.kit/bin/` if they exist:
- `fab` (shell dispatcher)
- `fab-go` (Go backend binary)
- `wt` (worktree manager)
- `idea` (backlog manager)

Keep `.gitkeep`.

Print: `Removed binaries from fab/.kit/bin/ (now provided by system shim).`

If none of the files exist, print: `fab/.kit/bin/ already clean, skipping.`

## Verification

1. `fab/project/config.yaml` contains `fab_version` field
2. `.envrc` does NOT contain `PATH_add fab/.kit/bin`
3. `fab/.kit/bin/` contains only `.gitkeep`
4. Running `fab --version` works (system shim is on PATH)
