### US-011: Create fab-review.md skill
Status: in_progress
Depends on: US-004

Description:
As a developer, I want the /fab:review skill so I can validate implementation against specs and checklists.

Acceptance Criteria:
- [ ] Create fab/.kit/skills/fab-review.md as comprehensive agent-agnostic markdown
- [ ] Must reference _context.md preamble at the top
- [ ] Checks: all tasks [x] in tasks.md, all checklist items verified in checklists/quality.md, run affected tests, features match spec requirements, no doc drift
- [ ] Re-reads each CHK-* item, inspects relevant code/tests, marks [x] or reports failure
- [ ] On pass: output ends with Next: /fab:archive
- [ ] On failure: presents options — fix code (/fab:apply with unchecked rework tasks), revise tasks (edit tasks.md then /fab:apply), revise plan (/fab:continue plan), revise specs (/fab:continue specs)
- [ ] For rework: unchecks tasks with <!-- rework: reason --> comment
- [ ] Updates .status.yaml: review: done on pass, review: failed on failure
- [ ] Context: config, constitution, tasks.md, checklists/quality.md, spec.md, centralized docs, relevant source code
- [ ] Example: review passes 10/12 checklist items, reports 2 failures with specific CHK IDs
- [ ] Negative case: if apply stage not complete, tell user to run /fab:apply first
