# docs-reorg-memory

## Summary

Analyzes memory files for themes and suggests reorganization. Read-only unless user approves changes. Identifies up to 10 themes, produces a read-only **Shape Report** (folders over the ~12 width bound / over depth 3 / under the ~5 floor; reserved domains `_shared`/`_unsorted` exempt), and proposes a reorg plan. Index files are regenerated by `fab memory-index`, never hand-edited.

> **Scope**: the Shape Report and split/merge/flatten migration kinds are **detect/diagnose only** in this skill today. The file-moving *apply* path (moving files into sub-domains + rewriting the relative links a move breaks) is a deferred follow-up.

## Flow

```
User invokes /docs-reorg-memory
│
├─ Pre-flight: docs/memory/index.md and domain files must exist
├─ Read: all memory files across all domains
├─ Diagnose: Shape Report (per-folder counts + depth vs ~5/~12/depth-3 bounds)
├─ (identify themes, propose reorganization: split/merge/flatten/move)
├─ (present plan, ask for approval)
│
└─ [if approved]
   ├─ Write/Edit: reorganized memory files (content moves only)
   └─ Bash: fab memory-index   (regenerates root + domain indexes deterministically)
```

### Tools used

| Tool | Purpose |
|------|---------|
| Read | All memory files and indexes |
| Write/Edit | Reorganized files (only with approval) |
| Bash | `fab memory-index` to regenerate indexes after approved moves |

### Sub-agents

None.
