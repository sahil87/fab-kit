# Migration: 0.46.0 to 0.47.0

## Summary

Relocate kit to system cache. Kit content now lives at `~/.fab-kit/versions/<version>/kit/` instead of `fab/.kit/` in each project. Hooks are inlined as `fab hook <subcommand>` commands.

## Pre-check

1. Confirm the project has a `fab/` directory — if not, skip this migration entirely
2. Run `fab kit-path` — if it exits non-zero, STOP with: `Cache not found. Run 'fab sync' first.`

## Changes

### 1. Inline hooks in `.claude/settings.local.json`

If `.claude/settings.local.json` exists:

1. Read the file and parse the `hooks` section
2. For each hook entry, check if the command matches the old-style pattern:
   - `bash "$CLAUDE_PROJECT_DIR"/fab/.kit/hooks/on-session-start.sh` -> `fab hook session-start`
   - `bash "$CLAUDE_PROJECT_DIR"/fab/.kit/hooks/on-stop.sh` -> `fab hook stop`
   - `bash "$CLAUDE_PROJECT_DIR"/fab/.kit/hooks/on-user-prompt.sh` -> `fab hook user-prompt`
   - `bash "$CLAUDE_PROJECT_DIR"/fab/.kit/hooks/on-artifact-write.sh` -> `fab hook artifact-write`
   - Also match the relative-path variant: `bash fab/.kit/hooks/on-*.sh`
3. Replace each matched command with the inline `fab hook <subcommand>` form
4. Write the updated settings file
5. Print: `Migrated: .claude/settings.local.json hooks ({N} entries inlined)`

If no old-style hooks found, print: `.claude/settings.local.json hooks already up to date.`

If `.claude/settings.local.json` does not exist, print: `No .claude/settings.local.json found, skipping hook migration.`

### 2. Remove `fab/.kit/` from project

If the directory `fab/.kit/` exists:

1. Delete the entire `fab/.kit/` directory
2. Print: `Removed: fab/.kit/ (kit content now served from system cache)`

If `fab/.kit/` does not exist, print: `fab/.kit/ already absent, skipping.`

### 3. Clean `.envrc`

If `.envrc` exists:

1. Read the file contents
2. If a line containing `PATH_add fab/.kit/scripts` exists, remove that line
3. Write the updated file
4. Print: `Cleaned: .envrc (removed PATH_add fab/.kit/scripts)`

If the line is not found, print: `.envrc already clean, skipping.`

### 4. Clean `.gitignore`

If `.gitignore` exists:

1. Read the file contents
2. Remove lines matching `fab/.kit/bin/*` and `!fab/.kit/bin/.gitkeep`
3. Write the updated file
4. Print: `Cleaned: .gitignore (removed fab/.kit/ entries)`

If no matching lines found, print: `.gitignore already clean, skipping.`

### 5. Remove Copilot code review config

If `.github/copilot-code-review.yml` exists:

1. Delete the file
2. Print: `Removed: .github/copilot-code-review.yml`

If `.github/copilot-code-review.yml` does not exist, print: `.github/copilot-code-review.yml already absent, skipping.`

### 6. Clean stale .gitignore entries

If `.gitignore` exists:

1. Read the file contents
2. Remove lines matching `/.ralph` and `fab/changes/**/.pr-done` if present
3. Write the updated file
4. Print: `Cleaned: .gitignore (removed stale entries)`

If neither line is found, print: `.gitignore already clean, skipping.`

### 7. Delete .pr-done files

Find and delete any `.pr-done` files under `fab/changes/`:

```bash
find fab/changes/ -name '.pr-done' -type f
```

If files are found:

1. Delete each `.pr-done` file
2. Print: `Deleted {N} .pr-done file(s) from fab/changes/`

If no `.pr-done` files are found, print: `No .pr-done files found under fab/changes/, skipping.`

## Verification

1. `fab/.kit/` does NOT exist in the project
2. `.claude/settings.local.json` hooks (if present) do NOT contain `fab/.kit/hooks/` references
3. `.envrc` does NOT contain `PATH_add fab/.kit/scripts`
4. `.gitignore` does NOT contain `fab/.kit/bin` entries
5. `.github/copilot-code-review.yml` does NOT exist
6. `.gitignore` does NOT contain `/.ralph` or `fab/changes/**/.pr-done` entries
7. No `.pr-done` files exist under `fab/changes/`
