# Migration: 2.1.6 to 2.2.0

## Summary

The `stage_directives` config key was removed from the kit's config surface. It
was promised by the scaffold and editable via `/fab-setup config`, but **no
skill or binary ever read it** — directives placed there were silently ignored.
The scaffold no longer seeds the block (its former defaults were redundant:
GIVEN/WHEN/THEN scenarios are mandated by `_generation.md`, and
`[NEEDS CLARIFICATION]` markers are intake-only since 1.10.0), and `/fab-setup`
no longer offers the `stage_directives` editor.

This migration drops the dead key from existing project configs. It also
defensively removes a `model_tiers` key if one is present (its dedicated
removal shipped in `0.21.0-to-0.22.0.md`; this is a no-op safety net for
configs where it reappeared).

`stage_hooks` is unrelated and MUST be preserved — it is live `fab status`
behavior (pre-hooks gate `start`, post-hooks run after `finish`).

## Pre-check

- `fab/project/config.yaml` exists. If not: skip all changes (nothing to
  migrate). Print: `Skipped: fab/project/config.yaml not present.`

## Changes

### 1. Remove the `stage_directives:` block

1. **If `config.yaml` does not contain a `stage_directives:` key**: this step
   is a no-op. Print: `Skipped: stage_directives already absent.`
2. **Otherwise**: remove the entire `stage_directives:` block (the key and all
   nested per-stage entries):

   ```bash
   yq -i 'del(.stage_directives)' fab/project/config.yaml
   ```

   If there are comment lines immediately before `stage_directives:` that
   describe it (e.g., `# Extra instructions the agent follows when generating
   each stage's artifact.`), remove those comment lines as well.

   Print: `Removed: stage_directives from fab/project/config.yaml.`

### 2. Remove a stray `model_tiers:` block (defensive)

1. **If `config.yaml` does not contain a `model_tiers:` key**: this step is a
   no-op. Print: `Skipped: model_tiers already absent.`
2. **Otherwise**: remove the block, including any immediately preceding
   descriptive comment line:

   ```bash
   yq -i 'del(.model_tiers)' fab/project/config.yaml
   ```

   Print: `Removed: model_tiers from fab/project/config.yaml.`

## Verification

- `yq '.stage_directives' fab/project/config.yaml` returns `null`
- `yq '.model_tiers' fab/project/config.yaml` returns `null`
- All other keys and their contents are unchanged — including `project`,
  `source_paths`, `checklist`, `fab_version`, and (when present)
  `true_impact_exclude`, `test_paths`, `agent`, `stage_hooks`
- Re-running this migration is a complete no-op (both steps print their
  `Skipped:` lines)
